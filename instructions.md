## PART 14 — KNOWN BUGS FIXED IN V2

1. **Pages showed `—` / blank on load** — Root cause: `U.init()` was called `onload` of the gapi script which is async/defer and may never fire. Fix: `update.js` now self-injects a guaranteed init via `DOMContentLoaded + setTimeout(300)`.

2. **Stale date calculations** — `daysUntil()` and `planMonth()` used `D.today` set at parse time. Fix: both now call `new Date()` fresh on every invocation.

3. **`S.bdt` / `S.monthly` were not reactive** — They were plain properties that had to be manually saved. Fix: converted to getter/setter pairs backed directly by localStorage. No more `S.save()` call needed.

4. **`flatCourses()` rebuilt on every call** — Fix: memoised with `_flatCache`. Cleared only on page load (PHASES never mutates at runtime).

5. **Phase progress bar fill class mismatch** — `ph-coursera` class was on the wrapper div not the fill div. Fix: fill divs now use inline `background:` style with `colMap[p.id]`.

6. **`U.Phase.planMonth()` was undefined** — `planMonth()` is a global function in `data.js`, not on `U.Phase`. Fix: added `planMonth()` proxy on `U.Phase`.

7. **`getCDone()` / `getMDone()` length mismatch crash** — If saved arrays had a different length than current data (after adding new courses/milestones), the app crashed. Fix: both functions now validate length and fall back to fresh defaults on mismatch.
